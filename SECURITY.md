# Security Policy

## Reporting a Vulnerability

If you discover a security issue in Kara, please do not open a public issue.

Use one of the following private channels:

- Email: sgonzalez@authuser.org
- GitHub Security Advisories: https://github.com/authuser-dev/karacron/security/advisories

Please include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Affected versions or environments
- Suggested remediation (if available)

## Response Timeline

- Initial acknowledgment: within 24 hours
- Triage and impact assessment: within 7 days
- Patch target (when confirmed): within 30 days
- Public disclosure: coordinated after a fix is available

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.x     | Yes       |
| < 1.0   | No        |

## Security Best Practices

- Keep Kara updated to the latest stable release
- Restrict local service access to trusted networks
- Use HTTPS/TLS for all remote integrations
- Rotate credentials and API keys regularly
- Review local logs and integration permissions periodically
